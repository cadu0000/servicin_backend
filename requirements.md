# Documento de Requisitos

## 1. Introdução

Este documento apresenta a especificação dos requisitos da ferramenta Serviçin.

## 2. Requisitos do Software

Tomando por base o contexto do sistema, foram identificados os seguintes requisitos de usuário:

### Requisitos Funcionais

| ID       | Descrição                                                                                                                                                                                                                                                       | Tipo    | Prioridade | Depende de               |
| :------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :------ | :--------- | :----------------------- |
| **RF01** | O sistema deve permitir o Cadastro de Prestadores solicitando e validando obrigatoriamente: Nome Completo, CPF ou CNPJ (único), Contatos, Eixos de Serviço, Valores por Serviço, Disponibilidade, Email (único) e Senha.                                        | Usuário | Alta       | RNF06                    |
| **RF02** | O sistema deve permitir o Cadastro de Solicitadores de Serviços solicitando e validando obrigatoriamente: Nome, CPF (único), Ocupação, Localização, Nome de Usuário (único) e Senha.                                                                            | Usuário | Alta       | RNF06                    |
| **RF03** | O sistema deve permitir o Login de Usuários (Prestador e Solicitador) mediante a entrada do Nome de Usuário e Senha cadastrados.                                                                                                                                | Usuário | Alta       | RF01, RF02 e RNF04       |
| **RF04** | O sistema deve permitir o Logout de Usuários a partir de qualquer página, encerrando a sessão ativa.                                                                                                                                                            | Usuário | Alta       | RF03                     |
| **RF05** | O sistema deve permitir que o usuário exclua permanentemente sua conta. Ao fazê-lo, o CPF associado deve ser desvinculado e o usuário deve receber uma confirmação da exclusão.                                                                                 | Usuário | Alta       | RF01 e RF02              |
| **RF06** | O sistema não deve permitir a criação de uma conta de usuário (prestador ou solicitador) se o CPF informado já estiver associado a uma conta existente.                                                                                                         | Usuário | Alta       | RF01                     |
| **RF07** | O sistema deve permitir que o Prestador cadastre um novo Serviço. As informações obrigatórias são: Eixo de Serviço, Nome do Serviço, Valor do Serviço e Localização de Atendimento.                                                                             | Usuário | Alta       | RF01                     |
| **RF08** | O sistema deve permitir que um Prestador associe múltiplos serviços (RF07) à sua conta.                                                                                                                                                                         | Sistema | Alta       | RF01                     |
| **RF09** | O sistema deve permitir a busca por serviços. Deve ser realizada procurando pelo nome do serviço ou eixo de serviço.                                                                                                                                            | Usuário | Alta       | RF07 e RF08              |
| **RF10** | O sistema deve permitir que o usuário refine a busca de serviços (RF09) utilizando filtros combináveis. Os filtros devem incluir: Nome do Prestador, Faixa de Valor do Serviço, Eixo do Serviço, Localização (Raio de KM), Tipo de Cobrança e Avaliação Mínima. | Usuário | Alta       | RF07 e RF08              |
| **RF11** | O sistema deve apresentar a lista de resultados correspondente após pesquisa realizada.                                                                                                                                                                         | Usuário | Alta       | RF07 e RF08              |
| **RF12** | O sistema deve permitir que um prestador de serviços cadastre um Novo Eixo de Serviço. As informações obrigatórias são: Nome do Eixo de Serviço e Descrição. O sistema deve garantir que o nome do eixo seja único.                                             | Usuário | Alta       | RF07                     |
| **RF13** | O sistema deve permitir que o Prestador defina se o Link de Contato via WhatsApp será exibido publicamente. Se ativado, o Solicitador deve poder iniciar uma conversa pelo link.                                                                                | Usuário | Média      | -                        |
| **RF14** | O sistema deve exibir a Agenda de um Prestador para um Solicitador. A agenda deve conter os horários de atendimento disponíveis e os horários/dias indisponíveis devido a agendamentos confirmados (exibindo apenas Data e Hora).                               | Usuário | Média      | RF07                     |
| **RF15** | O sistema deve permitir que usuários solicitadores de serviços agendem serviços com prestadores diretamente na plataforma. Definindo detalhamento do serviço a ser realizado, forma de pagamento e data e hora.                                                 | Usuário | Alta       | RF03, RF14, RF07 e RNF04 |
| **RF17** | O sistema deve permitir que o Prestador configure o modo de confirmação de agendamentos: (1) Aceite Automático ou (2) Aceite Manual (necessitando de confirmação em até 12 horas).                                                                              | Usuário | Alta       | RF03, RF14 e RF07        |
| **RF18** | A plataforma deve emitir alertas de confirmação de serviços agendados após confirmação de agendamento para ambos os usuários.                                                                                                                                   | Usuário | Alta       | RF03, RF14 e RF07        |
| **RF19** | Após a alteração do status do serviço para "Feito" (RF22), o sistema deve automaticamente alterar o status de pagamento para "Pagamento em aberto" e deve notificar o Solicitante a iniciar o processo de pagamento.                                            | Sistema | Alta       | RF15 e RF18              |
| **RF20** | O sistema deve gerenciar pagamentos por quatro métodos: Espécie (no ato do serviço), Cartão de Crédito, Cartão de Débito e PIX. Após a confirmação do pagamento, o status do serviço deve transitar para "Pago" (RF23).                                         | Sistema | Alta       | RF15 e RF18              |
| **RF21** | Quando o método de pagamento for Espécie, o sistema deve exigir a confirmação manual do Prestador para liberar a transição do status para "Pago".                                                                                                               | Sistema | Alta       | RF15 e RF18              |
| **RF22** | O sistema deve exibir os serviços contratados pelo solicitante, com as tags de: “Feito”, “Para Fazer”, “Em andamento”, “Cancelado”.                                                                                                                             | Sistema | Alta       | RF15, RF19 e RF20        |
| **RF23** | O sistema deve exibir o procedimento de pagamento com as tags: “Pago”, “Pagamento em aberto” e “Cancelado”.                                                                                                                                                     | Sistema | Alta       | RF15, RF19 e RF20        |
| **RF24** | O sistema deve permitir o cancelamento de serviços por parte dos usuários. Para isso, deve ser informado o motivo do cancelamento, após o cancelamento do serviço a agenda do prestador deve ser liberada automaticamente.                                      | Usuário | Alta       | RF15                     |
| **RF25** | O sistema deve permitir que o Solicitador avalie o serviço contratado somente após o status do serviço ser definido como "Feito" e "Pago". A avaliação deve ser dada em uma escala de 1 a 5 estrelas.                                                           | Usuário | Alta       | RF19 e RF22              |

### Regras de Negócio

| Identificador | Descrição                                                                                                                                                                                                                                                 | Prioridade | Depende de        |
| :------------ | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :--------- | :---------------- |
| **RN01**      | Todo usuário (Seja Prestador de Serviços ou Solicitador de Serviços) deve ser identificado com o CPF/CNPJ único associado a sua conta.                                                                                                                    | Alta       | RF01, RF02 e RF06 |
| **RN02**      | O Prestador de Serviços só exibirá avaliações caso finalize o serviço através da plataforma e realize a avaliação do solicitador.                                                                                                                         | Alta       | RF25              |
| **RN03**      | Todo usuário deve ser classificado como Prestador de Serviços ou Solicitador.                                                                                                                                                                             | Alta       | RF01, RF02        |
| **RN04**      | Para o cadastro de um Prestador, as seguintes informações serão obrigatórias: Nome completo, CPF ou CNPJ, Ocupação, Localização, Nome de usuário e Senha de acesso.                                                                                       | Alta       | RF01              |
| **RN05**      | O CPF (para solicitantes) e o CPF/CNPJ (para prestadores de serviços) deverá ser único na plataforma. O sistema não deve permitir o cadastro se o documento já estiver em uso.                                                                            | Alta       | RF01, RF02, RF06  |
| **RN06**      | O login no sistema só será permitido através de combinação válida entre usuário e senha de acesso.                                                                                                                                                        | Alta       | RF03              |
| **RN07**      | O nome de usuário deverá ser único para todos (prestadores ou solicitantes).                                                                                                                                                                              | Alta       | RF01, RF02        |
| **RN08**      | Um usuário autenticado poderá encerrar sua sessão (logout) a qualquer momento, invalidando seu acesso atual.                                                                                                                                              | Alta       | RF04              |
| **RN09**      | Um usuário poderá solicitar a exclusão de sua conta.                                                                                                                                                                                                      | Alta       | RF05              |
| **RN10**      | Uma confirmação de exclusão bem sucedida deve ser enviada ao usuário após a remoção de sua conta.                                                                                                                                                         | Alta       | RF05              |
| **RN11**      | Ao excluir uma conta, todos os dados do usuário devem ser removidos ou anonimizados, e o CPF deve ser liberado, permitindo que seja utilizado em um novo cadastro futuro.                                                                                 | Alta       | RF05              |
| **RN12**      | Apenas usuários do tipo "Prestador” poderão cadastrar eixos de serviços e Serviços.                                                                                                                                                                       | Alta       | RF07, RF12        |
| **RN13**      | Para cadastrar um novo Eixo de Serviço, é obrigatório informar nome e uma descrição.                                                                                                                                                                      | Alta       | RF12              |
| **RN14**      | O nome de um eixo de serviço deve ser único na plataforma, não podendo haver duplicatas.                                                                                                                                                                  | Alta       | RF12              |
| **RN15**      | Para cadastrar um novo Serviço, o Prestador deve informar: Eixo de Serviço, Nome do Serviço, Valor e localização do atendimento.                                                                                                                          | Alta       | RF07              |
| **RN16**      | Um prestador pode ter múltiplos serviços cadastrados e associados a sua conta.                                                                                                                                                                            | Média      | RF08              |
| **RN17**      | O prestador tem a opção de informar um link para contato pessoal. Se a opção não for marcada, o link não será visível.                                                                                                                                    | Média      | RF13              |
| **RN18**      | Os resultados das buscas por serviços devem ser refinados utilizando filtros na plataforma. Nome do Prestador, Faixa de Valor, Eixo do Serviço, Localização (definida por um raio em KM), Tipo de Cobrança e Avaliação Mínima (de 1 a 5 estrelas).        | Alta       | RF09, RF10        |
| **RN19**      | A agenda de um Prestador deve ser visível para os Solicitadores, mostrando claramente os horários disponíveis e indicando os horários já ocupados com agendamentos confirmados (sem expor detalhes do serviço ou do cliente, apenas a indisponibilidade). | Alta       | RF14              |
| **RN20**      | Apenas usuários do tipo solicitador podem agendar serviços.                                                                                                                                                                                               | Alta       | RF15              |
| **RN21**      | Ao realizar um agendamento, o Solicitador deve obrigatoriamente definir: o detalhamento do serviço, a forma de pagamento, a data e a hora desejada (dentre as disponíveis na agenda do prestador).                                                        | Alta       | RF15              |
| **RN23**      | Para agendamentos com "Aceite Manual", o Prestador tem um prazo de 12 horas para confirmar ou recusar a solicitação. Se nenhuma ação for tomada, o agendamento deve ser cancelado automaticamente.                                                        | Alta       | RF17              |
| **RN24**      | Após a confirmação dos agendamentos (seja manual ou automático) os usuários deverão receber uma notificação de confirmação.                                                                                                                               | Alta       | RF18              |
| **RN25**      | Um serviço agendado pode ter os seguintes status: "Para Fazer", "Em Andamento", "Feito" ou "Cancelado".                                                                                                                                                   | Alta       | RF22              |
| **RN26**      | Um pagamento associado a um serviço pode ter os seguintes status: "Pagamento em Aberto", "Pago" ou "Cancelado".                                                                                                                                           | Alta       | RF23              |
| **RN27**      | Quando o prestador alterar o status do serviço para “Feito” o sistema deve solicitar para o solicitador o pagamento e exibir o status de “Pagamento em aberto”.                                                                                           | Alta       | RF19              |
| **RN28**      | Tanto o prestador quanto o solicitador poderão cancelar um serviço agendado.                                                                                                                                                                              | Alta       | RF24              |
| **RN29**      | Ao solicitar o cancelamento, o usuário deverá, obrigatoriamente, informar o motivo.                                                                                                                                                                       | Alta       | RF24              |
| **RN30**      | Após o cancelamento de um serviço ser efetivado, o horário correspondente na agenda do Prestador deve ser liberado automaticamente, tornando-se disponível para novos agendamentos.                                                                       | Alta       | RF24              |
| **RN31**      | A avaliação consiste em uma nota de 1 a 5 estrelas.                                                                                                                                                                                                       | Alta       | RF25              |
| **RN32**      | Um Solicitador só pode avaliar um serviço após duas condições serem atendidas simultaneamente: o status do serviço estar como “Feito” E o status do pagamento estar como "Pago".                                                                          | Alta       | RF25              |
| **RN33**      | Os métodos de pagamento aceitos são: Espécie, Cartão de Crédito, Cartão de Débito e PIX.                                                                                                                                                                  | Média      | RF20              |
| **RN34**      | Se o método de pagamento escolhido for “Espécie” a mudança de status para "Pago” só ocorrerá após a confirmação manual do Prestador de que recebeu o valor.                                                                                               | Alta       | RF21              |

### Requisitos Não Funcionais

| Id.       | Descrição                                                                                                                                                                     | Categoria/Subcategoria     | Critério de Sucesso                                                                                                                                                             |
| :-------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **RNF01** | A plataforma Serviçin será desenvolvida e implementada exclusivamente como uma aplicação Web.                                                                                 | Restrição de implementação | O sistema não deve depender de instalação local ou de tecnologia que não seja nativamente suportada por navegadores web padrão.                                                 |
| **RNF02** | A interface de usuário (frontend) deverá ser totalmente funcional e visualmente consistente nos navegadores mais recentes do Google Chrome, Mozilla Firefox e Microsoft Edge. | Compatibilidade            | Os testes de regressão visual e funcional devem ser aprovados em 100% para as duas últimas versões do Chrome, Firefox e Edge.                                                   |
| **RNF03** | O sistema deve carregar a página inicial e de resultados em tempo hábil para o usuário.                                                                                       | Desempenho                 | O tempo de carregamento da página não deve exceder 3 segundos, sob condições seguras de rede.                                                                                   |
| **RNF04** | O sistema deve processar transações de login e agendamento de serviços instantaneamente.                                                                                      | Desempenho                 | O tempo do processamento das transações não deve exceder 3 segundos, sob condições seguras de rede.                                                                             |
| **RNF05** | O sistema deve manter estabilidade, mesmo com altas cargas de acesso.                                                                                                         | Desempenho                 | O sistema deve permitir que ao mesmo 100 usuários acessem a plataforma simultaneamente.                                                                                         |
| **RNF06** | O sistema deve garantir a proteção das credenciais do usuário (Nome de Usuário e Senha).                                                                                      | Segurança                  | As credenciais do usuário devem ser armazenadas utilizando criptografia e nunca texto simples.                                                                                  |
| **RNF07** | A interface do usuário deve ser intuitiva e de fácil manuseio para os usuários.                                                                                               | Usabilidade                | Os usuários devem conseguir completar o fluxo de atividades (cadastro, login, agendamentos e avaliações) de forma intuitiva, sem problemas e rápida (em um tempo de 3 minutos). |
| **RNF08** | O sistema deve ser responsivo e adaptar-se a diferentes dispositivos.                                                                                                         | Usabilidade                | O sistema deve permitir que usuários acessem por diferentes dispositivos e possam realizar as atividades sem impedimentos.                                                      |
| **RNF09** | O sistema deve permitir rápida correção de funcionalidades e incrementação, sem alterar sua estabilidade.                                                                     | Manutenção                 | Deve permitir o deploy de pequenas correções/implementações em produção.                                                                                                        |
| **RNF10** | A arquitetura deve ser projetada para acomodar o crescimento de funcionalidades futuras.                                                                                      | Manutenção                 | O sistema deve ser modularizado com um baixo acoplamento entre o frontend e backend.                                                                                            |
